# 系统功能汇总与需求分析报告

## 1. 分析结论

目标系统应定位为“手术室媒体控制与示教存档平台”，核心不是单点录制、单点会议或单台设备控制，而是围绕手术间、示教室、PC 和移动端建立统一的音视频路由、患者关联存档、会议协同、场景控制和运维审计能力。

从说明书基线看，MVP 必须复现并工程化以下能力：

- 视频路由和断开。
- PIP/PBP 多画面布局。
- 患者关联的视频录制、暂停、恢复、停止、静音和拍照。
- HIS 查询或手动新建患者。
- 视频会议、成员和音频控制。
- 文件查询、排序、回看、编辑、删除和下载。
- 场景编组、保存、应用和摄像头/音频控制。
- 手术室、示教室、PC 和移动端多终端访问。
- 系统设置、状态历史、维护自检和故障处理。

结合国内外同类系统，目标系统还应补足：

- 标准化医院接口和设备适配层。
- 更强的权限、审计、隐私和导出控制。
- 高可用存储、链路监控、容量告警和备份恢复。
- 发布、维护、培训和正式开放治理文档。

## 2. 功能域汇总

| 功能域 | 子功能 | MVP | 增强方向 |
| --- | --- | --- | --- |

| 视频路由 | 输入到输出、断开、预览、冲突检测 | 是 | 多策略路由、自动恢复、备用链路 |
| 多画面 | PIP、PBP、布局保存 | 是 | 自定义布局、跨屏联动 |

| 媒体资产 | 查询、排序、回看、编辑、删除、下载 | 是 | 审批流、水印、脱敏导出、全文索引 |
| 会议和示教 | 创建、加入、邀请、移除、关闭、音频控制 | 是 | 外部会诊网关、教学课件化 |

| 安全合规 | RBAC、审计、加密、备份、隐私 | 是 | 等保自动化证据、零信任接入 |

## 4. 需求分层

### 4.1 P0 必须项

- 角色和房间权限。
- 视频输入/输出配置和路由。
- PIP/PBP。
- 摄像头基础控制。
- 患者查询/新建。
- 录制、截图、文件归档和回放。
- 会议和示教基础能力。
- 场景保存和应用。
- 审计、告警、备份和基础安全。
- 部署、培训、试运行和回滚。

### 4.2 P1 应当项

- FHIR/HL7/DICOM 标准化接口。
- 外部会诊安全网关。
- 导出审批、水印和脱敏。
- 高可用部署和边缘降级。
- 链路质量、容量趋势和设备健康监控。
- 场景事务化回滚和版本管理。

### 4.3 P2 可选项

- 教学病例库、章节标记和课件化。
- 非诊断目的的转写、索引和视频质量分析。
- 多院区推广模板。
- 自动工单和预测性维护。

## 5. 与说明书基线的差距

| 说明书已有 | 工程化缺口 | 目标系统补充 |
| --- | --- | --- |
| 有登录，但权限细节少 | 角色、房间、操作级权限不足 | RBAC、统一身份、会话策略 |
| 有路由和 PIP/PBP | 缺少冲突、审计和回滚说明 | 幂等路由、冲突检测、审计 |
| 有录制和 U 盘下载 | 缺少保留、加密、导出审批 | 服务器优先、审批、水印、审计 |
| 有 HIS 搜索 | 接口标准和异常处理未展开 | 接口适配、重试、Mock、字段映射 |
| 有会议 | 外部访问安全边界不清 | 邀请制、会议权限、网关策略 |
| 有维护计划 | 缺少集中监控和告警指标 | 设备健康、链路质量、容量告警 |
| 有清洁消毒 | 软件开放文档不涉及感染控制联动 | 培训和现场 SOP 引用医院制度 |

## 6. MVP 验收摘要

正式开放前，至少完成以下端到端验收：

1. 手术室触控端完成登录、路由、PIP/PBP、摄像控制、录制、截图、场景应用。
2. 示教室端完成直播观看、会议加入、文档回看。
3. PC 端完成直播、会议、文档管理。
4. 受管移动端完成授权房间选择和限定功能操作。
5. HIS/EMR 患者查询或手动新建流程可用。
6. 媒体文件可按患者、日期、术式、医生、手术间检索。
7. 删除、下载、导出、会议外部访问、配置变更均可审计。
8. 断网、断电、存储不足、设备离线、服务重启等异常有告警和处理流程。
9. 用户培训和运维交接完成。
10. 追踪矩阵和正式开放检查清单全部通过。
