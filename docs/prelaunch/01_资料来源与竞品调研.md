## 2. 同类系统能力观察

### 2.1 国内方案常见侧重点

国内数字化手术室、手术示教和远程会诊方案通常围绕医院现有 HIS/EMR/PACS、手术室设备、院内专网和教学场景建设，常见能力包括：

- 多路视频采集、矩阵切换、手术直播和示教转播。
- 手术视频录制、截图、病历号/患者姓名/术式/医生等元数据归档。
- 手术示教室、会议室、移动查房或办公室观看。
- 与 HIS/EMR/PACS/手麻系统/排班系统对接。
- 院内私有化部署、国产化硬件适配、等保和院内网络分区。
- 手术间设备集中控制、场景预案和术前检查。

公开资料中常见的国内同类方案包括国内厂商 A 的数字化手术室方案、国内厂商 B 的数字化手术室/手术示教方案、国内厂商 C 的一体化手术室方案、国内厂商 D 的远程会诊/手术示教方案等。具体功能和资质需以采购阶段厂商正式技术白皮书、注册证、检测报告和医院现场需求为准。

### 2.2 国际方案常见侧重点

国际主流方案通常更强调基于 IP 的高质量视频网络、手术室集成控制、跨房间协作、可扩展设备生态和规范化影像/数据接口，代表性公开产品方向包括：

- 国际厂商 A 方案：一体化手术室集成、设备和媒体集中控制、手术室工作流协同。
- 国际厂商 B 方案：手术室集成、视频路由、设备控制和团队协作。
- 国际厂商 C 方案：内镜和手术室集成、图像/数据管理和工作流支持。
- 国际厂商 D 方案：数字化手术室媒体平台、IP 视频、录制、流媒体和协作。
- 国际厂商 E 方案：面向手术室的 IP 视频分发和低延迟高质量显示。
- 国际厂商 F 方案：手术室视频管理、显示、路由和集成组件。

这些系统的共同方向是：高可靠低延迟视频、开放或半开放接口、集中控制、跨终端协作、可审计的数据管理和更强的网络安全能力。

## 4. 需求结论

目标系统应以“手术媒体控制中台”为核心，而不是单一录制或会议工具。MVP 必须覆盖以下闭环：

1. 房间、设备、输入源、输出源可配置并可监测。
2. 术中可低延迟预览、路由、断开和切换。
3. 可按患者/手术/医生等元数据录制、截图、存档、检索和回看。
4. 可支持手术示教和远程会议，且权限、审计和授权明确。
5. 可保存场景模板，一键应用到摄像头、音频、布局和路由。
6. 可支撑医院内网私有化部署、安全审计、备份恢复和等保/医疗器械软件生命周期材料。
