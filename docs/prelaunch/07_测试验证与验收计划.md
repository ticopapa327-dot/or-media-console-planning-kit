# 测试验证与验收计划

## 1. 测试目标

验证系统满足 PRD/SRS、风险控制和上线要求，确认在医院现场环境中可支撑手术室媒体控制、示教、会议、存档和运维流程。

## 2. 测试范围

- 功能测试：登录、权限、路由、布局、录制、会议、文档、场景、设置、维护。
- 接口测试：IAM/存储/设备控制；HIS/EMR 患者信息接口暂不在当前 MVP 范围内。
- 性能测试：低延迟、并发观看、长时间录制、大文件检索。
- 稳定性测试：连续运行、每日重启、断网、断电、服务重启。
- 安全测试：权限绕过、导出、审计、漏洞扫描、弱口令。
- 可用性测试：触控操作、误触、告警提示、培训后操作成功率。
- 现场验收：手术室、示教室、PC、移动端和运维流程。

## 3. 测试环境

| 环境 | 用途 | 要求 |
| --- | --- | --- |
| DEV | 开发联调 | Mock 设备、合成患者数据、测试存储 |
| SIT | 系统集成 | 真实或仿真编码器、解码器、PTZ、会议服务 |
| UAT | 用户验收 | 接近生产拓扑，使用脱敏或模拟患者数据 |
| PROD-PILOT | 试运行 | 生产网络和真实设备，严格权限和审计 |

## 5. 性能验收

| 指标 | 验收目标 | 方法 |
| --- | --- | --- |
| 路由切换响应 | P95 小于 1 秒 | 自动化脚本和人工计时 |
| 手术室控制链路延迟 | P95 小于 300 ms | 现场链路测试 |
| 录制启动 | P95 小于 2 秒 | 连续 100 次启动测试 |
| 长时间录制 | 连续 8 小时无崩溃，文件可回放 | 稳定性测试 |
| 并发观看 | 按项目容量模型达标 | 压测和现场验证 |
| 检索响应 | 常用条件 P95 小于 3 秒 | 数据量模拟 |

## 6. UAT 验收流程

1. 培训用户并确认操作范围。
2. 使用脱敏或模拟患者数据完成术前、术中、术后全流程。
3. 临床用户完成操作评分和问题登记。
4. 信息科完成当前范围内接口、安全、备份和审计检查；HIS/EMR 患者信息接口不作为当前 MVP 验收项。
5. 设备工程师完成维护、自检、告警和故障演练。
6. 项目组归档 UAT 报告和遗留问题处理结论。

## 7. 缺陷分级

| 等级 | 定义 | 发布要求 |
| --- | --- | --- |
| P0 | 影响术中控制、数据安全、隐私泄露、系统不可用 | 必须清零 |
| P1 | 影响核心流程但有可接受临时方案 | 正式开放前清零或批准风险接受 |
| P2 | 影响体验或非核心功能 | 可带入计划但需记录 |
| P3 | 文案、轻微 UI、低风险优化 | 可排期 |

## 8. 发布准出

- P0 需求测试通过率 100%。
- P0/P1 缺陷清零。
- 安全扫描和权限测试通过。
- 风险控制措施已验证。
- 备份恢复和回滚演练通过。
- 用户手册、培训、运维和应急文档完成。
