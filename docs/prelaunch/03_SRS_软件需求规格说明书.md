# SRS：软件需求规格说明书

## 1. 系统范围

系统由手术室控制端、示教室控制端、PC 客户端、移动端、媒体与设备控制服务、会议服务、存档服务、接口服务、管理后台、审计与监控服务组成。系统运行在医院内网或经批准的私有云/专有云环境中。

## 2. 运行环境假设

- 手术室具备专用有线网络、交换机、编码器、解码器、显示器、摄像头、麦克风、扬声器和服务器存储。
- 关键媒体链路与医院办公网、访客网隔离。
- 外部会诊必须通过医院批准的安全边界、VPN、专线或网闸方案。
- 当前 MVP 暂不接入 HIS/EMR 获取患者信息；患者/手术信息使用手动录入或合成假数据。后续实际接入的 HIS/EMR/PACS/手麻系统接口由医院信息科确认。
- 移动端优先支持院内受管设备，不默认支持个人手机接入生产环境。

## 3. 功能需求

### 3.1 身份、房间和权限

| 编号 | 需求 | 优先级 | 验收 |
| --- | --- | --- | --- |
| FR-001 | 支持用户名/密码、LDAP/AD/OIDC 中至少一种身份源 | P0 | 可按医院身份源登录，失败有明确提示 |
| FR-001A | 支持角色权限：手术室操作员、示教用户、会诊专家、设备工程师、管理员、审计员 | P0 | 每个角色的菜单、房间和操作权限可配置 |
| FR-001B | 支持房间级授权和移动端房间选择 | P0 | 未授权用户无法查看或控制房间 |

### 3.2 视频源、输出和路由

| 编号 | 需求 | 优先级 | 验收 |
| --- | --- | --- | --- |
| FR-002 | 支持配置输入源类型：内镜、术野、全景、PACS 工作站、麻醉监护、会议输入、文件回放 | P0 | 管理端可新增、编辑、禁用和排序 |
| FR-003 | 支持可视化路由分配和断开 | P0 | 用户选择输入源和输出源后完成切换，界面显示连接成功或失败原因 |
| FR-003A | 支持路由冲突检测 | P0 | 输出源被占用或权限不足时阻止并提示 |
| FR-003B | 支持预览区显示实际视频信息 | P0 | 预览与输入源一致，离线时显示状态 |

### 3.3 画面布局和摄像控制

| 编号 | 需求 | 优先级 | 验收 |
| --- | --- | --- | --- |
| FR-004 | 支持 PTZ 摄像头缩放、方向和预置位 | P0 | 操作响应有反馈并写入审计 |
| FR-005 | 支持 PIP 大小屏和 PBP 四分屏 | P0 | 用户可选择输入源填入布局窗口并保存 |
| FR-005A | 支持布局模板管理 | P1 | 可创建、重命名、复制、删除和应用模板 |

| FR-006B | 预留后续手术排程导入 | P1 | 当前 MVP 不实现；现场接口确认后再进入开发 |

| 编号 | 需求 | 优先级 | 验收 |

## 4. 非功能需求

| 编号 | 分类 | 需求 | 目标 |
| --- | --- | --- | --- |
| NFR-001 | 性能 | 手术室控制链路低延迟 | 预览/控制 P95 小于 300 ms，现场硬件另行确认 |
| NFR-002 | 性能 | 路由切换响应 | P95 小于 1 秒 |
| NFR-003 | 可用性 | 核心服务高可用 | 关键服务支持主备或集群，单点故障有降级方案 |
| NFR-004 | 数据可靠性 | 录制文件完整性 | 断电/网络异常下可恢复或标记异常文件 |
| NFR-005 | 安全 | 传输和存储加密 | 管理流量 TLS；敏感元数据和密钥加密存储 |
| NFR-006 | 隐私 | 最小化患者信息 | 界面和导出按角色展示最少必要字段 |
| NFR-007 | 审计 | 审计不可篡改 | 审计日志集中保存，普通管理员不可删除 |
| NFR-008 | 易用性 | 触控可用 | 关键按钮适合戴手套操作，误触有确认 |
| NFR-009 | 兼容性 | 多分辨率和多终端 | 支持手术室大屏、嵌墙触控、PC 浏览器、受管平板 |
| NFR-010 | 可维护性 | 可观测性 | 日志、指标、链路质量、存储容量、设备状态可监控 |
| NFR-011 | 合规 | 医疗软件生命周期材料 | 需求、设计、风险、测试、发布记录可追踪 |

## 5. 外部接口需求

| 编号 | 接口 | 方向 | 最低要求 |
| --- | --- | --- | --- |
| INT-001 | HIS/EMR 患者查询 | 入站 | 当前 MVP 暂不支持；使用手动录入或合成假数据替代 |
| INT-002 | 手术排程 | 入站 | 当前 MVP 暂不支持；现场接口确认后再设计 |
| INT-003 | PACS/DICOM | 入站/出站 | 影像调阅、截图/关键帧归档需求待医院确认 |
| INT-004 | 身份认证 | 入站 | LDAP/AD/OIDC/SAML 至少一种 |
| INT-005 | 对象/文件存储 | 出站 | S3/NAS/SMB/NFS 视医院环境确定 |
| INT-006 | 设备控制 | 双向 | ONVIF/厂商 SDK/串口/网络控制协议 |
| INT-007 | 会议网关 | 双向 | WebRTC/SIP/RTMP/SRT 视部署确定 |

## 6. 数据需求

系统至少应管理以下数据对象：用户、角色、房间、设备、输入源、输出源、路由会话、布局模板、患者、手术、录制任务、媒体文件、会议、场景、告警、维护记录、审计日志、系统配置。

患者和媒体数据必须支持保留期限、归档、删除审批、导出审批和审计追踪。正式生产数据迁移前必须完成数据字典冻结。
