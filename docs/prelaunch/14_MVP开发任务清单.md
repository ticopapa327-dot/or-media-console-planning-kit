# MVP 开发任务清单

版本：0.1
日期：2026-06-29
输入资料：PRD、SRS、系统架构、测试计划、设备清单与电气连接设计。
目标：把已形成的需求和现场拓扑资料转化为第一阶段可开发、可测试、可验收的任务包。

## 1. MVP 范围原则

MVP 不追求一次覆盖完整医院生产系统，而是先做出可演示、可联调、可扩展的核心闭环：

- 以房间、设备、端口、连接、视频源、显示端、服务器和存储为核心数据模型。
- 先支持标准版单手术室、示教报告厅、远程示教端和双服务器架构。
- 真实硬件接入先通过适配层隔离，第一版允许使用模拟设备和模拟媒体流验证业务流程。
- 所有高风险操作必须进入审计日志，所有 P0 功能必须能映射到测试用例。
- 原始图纸和厂商资料不进入公开仓库，公开仓库只保留脱敏后的工程模型和方法。

| 状态 | 含义 |

| Backlog | 已识别，尚未进入近期开发 |

## 5. GitHub Issue 转换建议

后续可将每个 `MVP-xxx` 转为一个 GitHub Issue，标题格式建议：

```text
[MVP-004] 实现房间、设备、端口和连接模型
```

建议标签：

| 标签 | 用途 |
| --- | --- |
| `area:frontend` | 前端界面和交互 |
| `area:backend` | 后端 API、权限、业务逻辑 |
| `area:media` | 视频路由、录制、流媒体、会议 |
| `area:device` | 设备模型、适配器、现场拓扑 |
| `area:test` | 自动化测试、验收测试、测试数据 |
| `priority:p0` | MVP 必须完成 |
| `priority:p1` | MVP 可延后但需要预留接口 |

## 6. 当前阻塞项

| 阻塞项 | 影响 | 建议动作 |
| --- | --- | --- |
| 真实设备协议和 SDK 未确认 | 影响矩阵、编码板、PTZ、音频控制真实接入 | MVP 使用模拟适配器，同时准备协议调研清单 |
| HIS/EMR/PACS 接口未确认 | 影响患者和手术排程自动关联 | 当前 MVP 暂不接入，使用手动录入和合成假数据 |
| 存储容量和保留期限未确认 | 影响录制策略和归档策略 | 先按可配置参数实现容量估算和告警 |
| 网络隔离和远程访问边界未确认 | 影响远程示教和移动端范围 | 默认院内局域网，公网访问不进入 MVP |
