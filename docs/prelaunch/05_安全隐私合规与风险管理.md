# 安全隐私合规与风险管理

## 1. 合规边界

本系统涉及手术音视频、患者身份信息、手术信息、操作日志和设备状态，属于高敏感医疗业务系统。默认产品声明不包含诊断或治疗功能。如果未来增加 AI 识别、临床决策支持、治疗建议或自动控制手术设备，必须重新评估医疗器械属性、风险等级、临床评价、注册检测和质量体系要求。

## 3. 数据分类

| 数据 | 分类 | 控制要求 |
| --- | --- | --- |
| 患者姓名、病历号、手术信息 | 敏感个人信息/医疗数据 | 最小展示、加密、访问审计、脱敏导出 |
| 手术视频和图片 | 敏感医疗媒体 | 授权访问、保留期限、导出审批、水印 |
| 会议音视频 | 敏感业务数据 | 加密传输、会议权限、禁止未授权录制 |
| 操作日志 | 安全审计数据 | 集中保存、不可篡改、定期审查 |
| 设备状态和网络拓扑 | 运维敏感数据 | 管理员可见、禁止外泄 |

## 5. 隐私原则

- 明确用途：手术示教、会诊、存档、质控和培训，禁止超范围使用。
- 最小必要：非必要不显示患者完整姓名、病历号和身份证件信息。
- 授权访问：按角色、房间、手术和教学用途授权。
- 可追溯：谁在何时查看、下载、删除、分享都应可审计。
- 可删除和保留：按医院制度和法规确定保留期限；到期归档或删除需留痕。
- 数据出境：默认不出境；跨境会诊或云服务必须单独评估。

## 7. 医疗软件生命周期要求

### 7.1 需求和追踪

- 所有 P0/P1 需求必须有来源、设计、测试和验收记录。
- 安全相关需求必须纳入风险控制和回归测试。
- 需求变更必须评估对法规、风险、接口、测试和培训材料的影响。

### 7.2 设计和实现

- 关键控制链路采用防御式设计：幂等、超时、重试、失败提示、回滚。
- 厂商 SDK 和硬件控制隔离在适配层。
- 所有接口定义版本化，避免现场接口变更造成隐性故障。

### 7.3 验证和确认

- 验证：确认软件按规格实现。
- 确认：确认临床/教学/运维场景满足用户实际需要。
- 现场确认必须覆盖手术室、示教室、PC、移动端、HIS 接口、存储和异常场景。

### 7.4 发布和维护

- 每次发布必须包含版本说明、已知问题、回滚方案、数据库迁移脚本、配置变更说明和安全扫描报告。
- 生产缺陷按严重度分级处理；涉及患者隐私或术中安全的缺陷需启动事件响应。
