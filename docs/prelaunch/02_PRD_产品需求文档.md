# PRD：数字一体化手术室媒体控制系统

## 1. 产品概述

暂定产品名：OR Media Console Next  
产品形态：院内部署的数字化手术室媒体控制、示教、会议和媒体资产管理平台  
目标用户：手术室医护、示教室教师、远程会诊专家、设备工程师、信息科、质控和系统管理员

产品愿景：把手术室多源音视频、设备控制、患者关联媒体存档和远程示教统一到可审计、可维护、可扩展的平台中，减少术中切换复杂度，提升教学协同和术后资料管理效率。

## 2. 产品边界

### 2.1 包含范围

- 多房间、多终端、多输入输出源管理。
- 视频路由、PIP/PBP、多画面布局和摄像 PTZ 控制。
- 患者/手术信息关联的视频录制、截图、暂停、恢复、停止、静音、存储和回看。
- 视频会议、手术示教、直播观看和会议成员管理。
- 文档/媒体资产查询、排序、编辑、删除、下载和审计。
- 场景预设、编组、保存、应用和设备联动。
- 系统设置、状态历史、告警、维护计划和自检。
- 身份认证、存储、会议平台和设备接口；HIS/EMR/PACS 患者信息接口暂不进入当前 MVP。
- 安全、隐私、审计、备份恢复和上线运维能力。

### 2.2 不包含范围

- 不提供诊断或治疗结论。
- 不做自动手术指导或自主控制手术设备。
- 不在 MVP 中提供 AI 病灶识别、手术风险预测等临床决策能力。
- 不在当前 MVP 中从 HIS/EMR 获取患者信息；患者和手术信息使用手动录入或合成假数据。
- 不默认支持公网开放访问；外部会诊必须经医院网络和安全审批。

## 3. 用户画像

| 角色 | 目标 | 痛点 | 关键需求 |
| --- | --- | --- | --- |
| 主刀/巡回护士 | 术中快速切换画面、录制资料 | 设备多、切换慢、误操作风险 | 大触控、低延迟、场景一键应用、权限保护 |
| 示教室教师 | 观看手术、互动讲解、调取资料 | 多房间信号难管理 | 房间直播、会议互动、媒体回放、标注说明 |
| 会诊专家 | 远程参与会议 | 音视频质量和安全边界不清 | 加密会议、授权加入、清晰音视频 |
| 设备工程师 | 维护设备状态 | 故障定位难、日志分散 | 设备健康、告警、维护计划、自检报告 |
| 信息科/安全 | 接口和数据合规 | 患者隐私、权限、审计、备份 | RBAC、日志、加密、接口标准化、等保 |
| 管理员/质控 | 资料检索和追溯 | 文件命名混乱、缺少追踪 | 统一元数据、检索、报表、留存策略 |

## 5. MVP 功能需求

| 编号 | 需求 | 验收标准 | 来源 |
| --- | --- | --- | --- |
| FR-001 | 系统应支持多手术间、示教室、PC 和移动终端登录，并按角色展示功能 | 不同角色登录后只能看到授权房间和授权功能 | 说明书/竞品 |
| FR-002 | 系统应维护视频输入源、输出源、房间、设备和在线状态 | 管理员可配置源和输出，终端显示在线/离线状态 | 说明书/竞品 |
| FR-003 | 系统应支持将任一授权输入源路由到任一授权输出源 | 路由成功后输出端显示输入源名称和连接状态，可断开 | 说明书 |
| FR-004 | 系统应支持术野/全景等摄像头的缩放和云台控制 | 用户可在预览区控制，操作记录进入审计日志 | 说明书 |
| FR-005 | 系统应支持 PIP 和 PBP 多画面布局 | 至少支持大小屏和四分屏，并可保存布局 | 说明书 |
| FR-006 | 当前 MVP 应支持手动新建患者/手术信息，并可使用合成假数据完成录制关联 | 手动录入字段可校验；不依赖 HIS/EMR 查询 | 当前约束 |
| FR-007 | 系统应支持视频录制、暂停、恢复、停止、静音/取消静音和拍照 | 录制状态有明显提示；文件与患者/手术信息自动关联 | 说明书 |
| FR-008 | 系统应支持服务器存储和受控移动介质导出 | 默认存储到服务器；移动导出受权限、格式和审计限制 | 说明书/安全 |
| FR-009 | 系统应支持视频会议创建、加入、邀请、移除、退出、关闭和音频控制 | 会议创建者可关闭会议；成员权限可配置 | 说明书 |
| FR-010 | 系统应支持媒体文件查询、排序、信息查看、回看、编辑、删除和下载 | 查询条件包括用户、病历号、姓名、日期、术式、医生、手术间 | 说明书 |
| FR-011 | 系统应支持场景编组、场景保存、重命名、应用和冲突提示 | 应用场景前显示影响设备，失败时可回滚或提示 | 说明书/竞品 |
| FR-012 | 系统应支持系统基本信息、语言、录制码率、版本和开机时长查看 | 管理员可查看版本和参数，普通用户仅查看授权信息 | 说明书 |
| FR-013 | 系统应记录状态历史、告警和关键操作审计 | 登录、路由、录制、删除、下载、会议、配置变更均可追踪 | 说明书/合规 |
| FR-014 | 系统应支持维护计划、自检和故障知识库 | 每次开机执行自检；维护记录可导出 | 说明书 |

## 7. 关键指标

| 指标 | 目标值 |
| --- | --- |
| 术中路由切换响应 | P95 小于 1 秒，设备链路允许时 |
| 本地预览端到端延迟 | 手术控制链路 P95 小于 300 ms；示教直播按协议另定 |
| 录制启动时间 | P95 小于 2 秒 |
| 系统可用性 | 生产环境月可用性不低于 99.5%，关键手术间目标 99.9% |
| 审计完整性 | 高风险操作 100% 留痕 |
| 权限绕过 | 0 个开放缺陷 |
| 严重数据丢失 | 0 次 |

## 8. 上线成功标准

- P0 功能需求 100% 完成并通过验收。
- P0/P1 安全缺陷清零，P2 缺陷有风险接受记录。
- 身份认证、存储等当前范围内现场接口完成联调；HIS/EMR/PACS 患者信息接口不作为当前 MVP 开放门禁。
- 完成至少 2 周试运行，覆盖手术室、示教室、PC 和移动场景。
- 完成用户培训、管理员培训、应急演练和回滚演练。
