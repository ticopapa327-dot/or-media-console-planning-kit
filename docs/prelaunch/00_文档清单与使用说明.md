# 数字一体化手术室媒体控制系统正式开放前文档清单

版本：0.1  
日期：2026-06-28  
适用阶段：立项评审、需求评审、架构评审、开发实施、验证确认、试运行和正式开放评审  
主要输入：已脱敏的标准版数字化手术室说明资料。原始 PDF 不随公开仓库发布。

## 1. 文档包目标

本文件夹汇总开发同类型数字化手术室媒体控制软件在正式开放前应具备的核心文档。文档按软件工程生命周期组织，并结合 Codex 官方手册建议的工作方式：用清晰任务上下文、`AGENTS.md` 持久化项目规则、测试和评审闭环、MCP/技能沉淀可复用流程、安全权限边界来提升研发可靠性。

## 3. 使用方式

1. 先阅读 `01_资料来源与竞品调研.md` 和 `02_PRD_产品需求文档.md`，确认产品边界和 MVP。
2. 需求评审后冻结 `03_SRS_软件需求规格说明书.md` 的 P0/P1 需求。
3. 架构评审时同步确认 `04_系统架构与接口设计.md`、`05_安全隐私合规与风险管理.md`。
4. 每个迭代结束更新 `09_追踪矩阵与开放检查清单.md`，保证需求、测试、风险和发布项闭环。
5. 正式开放前，逐项完成 `09_追踪矩阵与开放检查清单.md` 的 Release Gate。

## 4. 重要前提

- 本文档包是软件项目计划与需求分析，不构成医疗器械注册结论。
- 系统默认不用于诊断或治疗决策；若产品声明、算法功能或临床使用范围发生变化，必须重新开展法规分类、临床评价和风险管理。
- 国内法规、医院接口规范、数据保留期限和院内网络边界必须在项目落地医院现场确认。
- 竞品资料仅用于能力对标和需求启发，不复制界面、文案、专利设计或受保护实现。
