# 项目计划、WBS 与里程碑

## 0. 项目章程摘要

| 项目 | 内容 |
| --- | --- |
| 项目名称 | 数字一体化手术室媒体控制系统 |
| 发起目标 | 建设可在医院内网运行的手术室媒体控制、示教、会议、存档和运维平台 |
| 业务价值 | 降低术中音视频切换复杂度，提升手术示教、资料归档、质控追溯和设备运维效率 |
| 主要范围 | 视频路由、PIP/PBP、录制截图、会议示教、文档管理、场景控制、接口集成、安全审计、部署运维 |
| 不在范围 | 诊断或治疗决策、自动手术指导、未经批准的公网访问、MVP 阶段的临床 AI |
| 关键成功标准 | P0 功能验收通过，安全和隐私风险受控，试运行问题闭环，正式开放检查清单通过 |
| 主要干系人 | 手术室、示教室、信息科、设备科、医务/质控、法规/质量、供应商和项目团队 |

## 1. 项目目标

在 8 到 10 个月内完成可在单院区试运行的数字一体化手术室媒体控制系统 MVP，并具备正式开放前所需的需求、设计、风险、测试、部署、培训和运维文档。

## 2. 团队角色

| 前端/客户端 | 触控端、PC、移动端、管理后台 |

| 测试工程师 | 测试计划、自动化、性能、安全、现场验收 |

## 3. 阶段计划

| 阶段 | 周期 | 目标 | 主要交付物 |
| --- | --- | --- | --- |
| P0 立项与调研 | 第 1-3 周 | 明确范围、竞品、法规和现场假设 | 调研报告、项目章程、初版 PRD |
| P1 需求与原型 | 第 4-8 周 | 冻结 MVP 需求和关键流程 | PRD、SRS、原型、验收标准 |
| P2 架构与技术预研 | 第 6-10 周 | 验证视频、会议、设备和接口方案 | 架构设计、PoC、接口清单 |
| P3 MVP 开发迭代 | 第 11-24 周 | 完成 P0 功能 | 可运行系统、单元/集成测试 |
| P4 集成与系统测试 | 第 25-30 周 | 完成硬件、HIS、存储、会议联调 | 系统测试报告、缺陷清单 |
| P5 试运行 | 第 31-36 周 | 小范围真实流程试运行 | 试运行报告、培训记录、问题闭环 |
| P6 正式开放准备 | 第 37-40 周 | 完成上线评审和开放 | 发布包、回滚方案、开放检查单 |

## 6. 主要依赖

- 现场设备清单、厂商 SDK、网络拓扑和手术室施工条件。
- HIS/EMR/PACS/身份认证接口授权和联调窗口。
- 医院信息安全、等保和数据治理要求。
- 存储容量预算、备份策略和保留期限。
- 临床科室试运行排期和培训窗口。

## 7. 项目风险

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| 厂商设备协议封闭 | 影响控制和路由能力 | 早期 PoC、适配层、采购协议写入接口要求 |
| 现场接口延期 | 影响患者关联和排程 | 提前申请联调，提供 Mock 和手动录入降级 |
| 网络环境不满足低延迟 | 影响术中体验 | 现场网络评估、专网/VLAN、边缘节点 |
| 存储容量低估 | 影响录制 | 按码率和保留期限测算，容量告警和归档 |
| 合规边界不清 | 影响开放 | 用途声明审查，隐私和安全评审前置 |
